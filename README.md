# khaaliNames

khaaliNames is an onchain utility to generate unique usernames for dApps by combining words from 
onchain dictionaries. This project is inspired by the 
[unique-names-generator](https://www.npmjs.com/package/unique-names-generator) npm package.

## How it works

Names are generated by combining words from three dictionaries (colors, adjectives, and animals) 
with a numeric suffix ~ _bold-blue-fox-3_. The utility also has an opinionated milestone system to 
track number of signups for a dApp.

### NameType

The following name types are present with varying uniqueness:

| NameType | Format | Unqiue Combinations |
| :------: | :----: | :-----------------: |
| COLOR | blue-n | 50 * n |
| ANIMAL | fox-n | 350 * n |
| ADJECTIVE | bold-n | 1200 * n |
| COLOR_ANIMAL | blue-fox-n | 17_500 * n |
| ADJECTIVE_ANIMAL | bold-fox-n | 420_000 * n |
| ADJECTIVE_COLOR_ANIMAL | bold-blue-fox-n | 21_000_000 * n |
| COLOR_ADJECTIVE_ANIMAL | blue-bold-fox-n | 21_000_000 * n |

### Milestone

Milestones map to opinionated (NameType, n) pairs to track user growth

| Milestone | Format | Unique Combinations | Significance |
| :-------: | :----: | :-----------------: | :----------: |
| ANIMAL_30 | animal-n (`n` in 1-30) | ~10_500 | first ~10k users |
| COLOR_ANIMAL_5 | color-animal-n (`n` in 1-5) | ~87_500 | first ~100k users |
| ADJ_ANIMAL_2 | adjective-animal-n (`n` in 1-2) | ~840_000 | first ~1M users |
| ADJ_COLOR_ANIMAL_3 | adjective-color-animal-n (`n` in 1-3) | ~49_000_000 | first ~50M users |

..and so on..

The milstone based names help us quickly identify who were early users and reward them accordingly.

### Dictionary Storage

Onchain dictionaries are deploying using solady's 
[SSTORE2](https://github.com/Vectorized/solady/blob/main/src/utils/SSTORE2.sol). Each word is 
packed into `bytes16` which allows for ~1500 words per data contract (to allow for a 24KB contract 
size limit). The decision to deploy words as contract bytecode significantly improves gas 
requirements while deploying, while adds a marginal cost to reading;

- Write: 22_100 gas per storage slot vs 200 gas/byte (6_400 gas for 32 byte storage)
- Read: ~2_100 gas for `SLOAD` vs ~2_600 gas for `EXTCODECOPY`

### Gas Consideration
> Gas report for 2000 optimizations can be found in [`gas-report.txt`](./gas-report.txt) \
> For different optimizer runs see [`reports/`](./reports/)

Runtime gas for `getRandomName` (median, optimizer runs = 2000):

| Function | Gas |
| -------- | --- |
| Milestone-based (e.g., `ANIMAL_30`) | 12_711 |
| NameType-based (e.g., `ADJECTIVE_COLOR_ANIMAL`) | 12_736 |

For deployemnt, `5_266_515` (~5M) gas was used; Please see deployment information in 
[`broadcast/`](./broadcast/DeployNamesV1.s.sol/31/run-latest.json) for more information.

## Quick Start

There are two ways to use this onchain utility, directly onchain, or to deploy yourself.

### Direct Usage

The contracts are currently deployed to RSK Testnet. The main contract to query is `khaaliNamesV1`
at [`0x0325F6f793525D18cfb665607031f3DC24092632`](https://explorer.testnet.rootstock.io/address/0x0325F6f793525d18CFb665607031f3Dc24092632).

There are two functions that can be called,

- `getRandomName(address recipient, Milestone milestone)`
- `getRandomName(address recipient, NameType nameType, uint8 n)`

..where the two enums are,

```sol
enum NameType {
    NONE,                    // 0 - no-op, reverts
    ANIMAL,                  // 1 - 350 unique names
    COLOR,                   // 2 - 50 unique names
    ADJECTIVE,               // 3 - 1200 unique names
    COLOR_ANIMAL,            // 4 - 50 * 350 = 17_500 unique names
    ADJECTIVE_ANIMAL,        // 5 - 1200 * 350 = 420_000 unique names
    ADJECTIVE_COLOR_ANIMAL,  // 6 - 1200 * 50 * 350 = 21_000_000 unique names
    COLOR_ADJECTIVE_ANIMAL   // 7 (non-native) - 21_000_000 unique names
}

enum Milestone {
    NONE,                // 0 - no-op, reverts
    ANIMAL_30,           // n in [1,30] :: first ~10k users
    COLOR_ANIMAL_5,      // n in [1,5]  :: next ~90k users
    ADJ_ANIMAL_2,        // n in [1,2]  :: next ~900k users
    ADJ_COLOR_ANIMAL_3   // n in [1,3]  :: next ~49m users
}
```

### Self Deployment

For self deployment, you should follow the following steps,

1. `cp .env.example .env`
2. Fill out the required values (you need an api_key from [rootstock](https://dashboard.rpc.rootstock.io/login))
3. `forge build`
4. `forge test`
5. Run the following command to deploy
```sh
forge script script/DeployNamesV1.s.sol:DeployNamesV1 \
--rpc-url rsk_testnet \
--legacy \
--broadcast
```
6. To verify the contracts, you need to grab the address from `deployments.json` and run the following,
```sh
# Run this to verify all dictionary contracts
forge verify-contract \
--constructor-args $(cast abi-encode "constructor(address)" <data-contract>) \
--chain-id 31 --watch --compiler-version v0.8.28 \
--verifier custom \
--verifier-url https://be.explorer.testnet.rootstock.io/api/v3/etherscan \
<dictionary-address> \
<dictionary-file-path>:<dictionary-contract-name>

# Run this to verify main utility contract
forge verify-contract \
--constructor-args $(cast abi-encode "constructor(address,address,address)" <animal-dictionary-contract-addr> <color-dictionary-contract-addr> <adjective-dictionary-contract-addr>) \
--chain-id 31 --watch --compiler-version v0.8.28 \
--verifier custom \
--verifier-url https://be.explorer.testnet.rootstock.io/api/v3/etherscan \
<khaaliNames-contract-address> \
src/khaaliNamesV1.sol:khaaliNamesV1
```
